**Tests**: Not required — manual review only (verify renders in terminal and GitHub).

## Work Package WP01: Create End-to-End Workflow Cheatsheet (Priority: P1) MVP

**Goal**: Create `docs/workflow-cheatsheet.md` containing the complete spec-kitty → kasmos lifecycle reference, covering all phases, commands, orchestration modes, daily session patterns, and prerequisites. Update existing docs with cross-links.
**Independent Test**: Give the cheatsheet to an operator unfamiliar with the command sequence; they can identify the correct command for each phase within 10 seconds of scanning.
**Prompt**: `/tasks/WP01-create-workflow-cheatsheet.md`
**Estimated prompt size**: ~350 lines

### Included Subtasks
- [x] T001 Create `docs/workflow-cheatsheet.md` with document header and prerequisites section
- [x] T002 Write end-to-end pipeline overview with visual flow diagram
- [x] T003 Write phase reference cards for spec-kitty planning phases (specify, clarify, plan, research, tasks, analyze)
- [x] T004 Write phase reference cards for execution phases (implement, launch, monitor, review, accept, merge)
- [x] T005 Write kasmos orchestration sub-workflow section (both modes, FIFO summary, recovery actions)
- [x] T006 Write daily session quick-reference section
- [x] T007 Add cross-references section and update existing docs with back-links

### Implementation Notes
- Write subtasks sequentially (T001→T007) since each section builds on the document structure.
- Source all content from existing codebase artifacts — no external research needed.
- Reference `docs/cheatsheet.md` for kasmos FIFO commands (don't duplicate the full table — summarize and link).
- Reference `docs/getting-started.md` for Zellij navigation (link, don't duplicate).
- Keep each phase reference card to a consistent format: command, description, input, output, flags.
- Mark optional phases clearly with `(optional)` labels.

### Parallel Opportunities
- None — all subtasks write to the same file sequentially.

### Dependencies
- None (first and only work package).

### Risks & Mitigations
- **Stale content**: Commands or flags may change. Mitigation: source from actual CLI definitions in `crates/kasmos/src/main.rs` and spec-kitty init output.
- **Overlap with existing cheatsheet.md**: Mitigation: workflow-cheatsheet focuses on lifecycle flow; kasmos cheatsheet focuses on CLI/FIFO detail. Cross-link between them.
