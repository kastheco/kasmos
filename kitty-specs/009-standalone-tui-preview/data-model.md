## State Machine: Animation Loop

The animation loop advances one WP per tick (~3 seconds), following this deterministic state machine:

```
                    ┌──────────┐
                    │ Pending  │
                    └────┬─────┘
                         │ set started_at
                         v
                    ┌──────────┐
              ┌─────│  Active  │─────┐
              │     └──────────┘     │
              │ tick % 7 == 0        │ tick % 7 != 0
              │ (~14.3%)             │ (~85.7%)
              v                      v
        ┌──────────┐          ┌───────────┐
        │  Failed  │          │ ForReview │
        └────┬─────┘          └─────┬─────┘
             │ next tick:           │ set completed_at,
             │ retry (set           │ completion_method
             │ started_at)          v
             │                ┌───────────┐
             └───────────────>│ Completed │
                  (via Active)└───────────┘
```

### Wave State Derivation

After each WP state change, recompute wave states:

| Condition | Wave State |
|-----------|------------|
| All WPs in wave are `Pending` | `WaveState::Pending` |
| Any WP is `Active` or `ForReview` | `WaveState::Active` |
| All WPs are `Completed` | `WaveState::Completed` |
| Any WP is `Failed` and no WP is `Active` | `WaveState::PartiallyFailed` |

### Cycle Reset

When all WPs reach `Completed`:
1. Set `run.state = RunState::Completed`, `run.completed_at = Some(now)`
2. Broadcast via `watch_tx`
3. Sleep 2 seconds
4. Reset all WPs to initial states (clone of `initial_run`)
5. Broadcast reset state
6. Continue loop
