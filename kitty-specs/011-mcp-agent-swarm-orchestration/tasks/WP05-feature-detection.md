# WP05: Feature Detection & Resolution

## Objective

Implement feature detection logic used by both the `kasmos [spec-prefix]` launcher and the `infer_feature` MCP tool. Detection tries multiple strategies in order: CLI argument → git branch → current directory → None (trigger selector). This is a foundational utility that enables feature-scoped sessions.

## Context

### Current State

`feature_arg.rs` already exists with some feature argument resolution logic. It needs to be refactored into the new `launch/detect.rs` module while maintaining backward compatibility for `kasmos list` and `kasmos status`.

### Target Architecture

```
launch/detect.rs
├── detect_feature(spec_prefix: Option<&str>) -> Result<Option<FeatureInfo>>
├── detect_from_arg(prefix: &str) -> Result<Option<FeatureInfo>>
├── detect_from_branch() -> Result<Option<FeatureInfo>>
├── detect_from_directory() -> Result<Option<FeatureInfo>>
└── list_features() -> Result<Vec<FeatureInfo>>

    pub dir: PathBuf,           // absolute path to kitty-specs/<slug>/
    pub has_spec: bool,
    pub has_plan: bool,
    pub has_tasks: bool,
}
```

### T029: Create `launch/detect.rs` with `detect_feature()`

**Purpose**: Main entry point that tries detection strategies in priority order.

**Steps**:

1. Create `crates/kasmos/src/launch/detect.rs`:
   ```rust
   //! Feature detection and resolution.
   //!
   //! Strategies tried in order: CLI arg → git branch → directory → None.
   
   use anyhow::Result;
   use std::path::{Path, PathBuf};
   
   /// Information about a detected feature.
   #[derive(Debug, Clone)]
   pub struct FeatureInfo {
       /// Feature slug (e.g., "011-mcp-agent-swarm-orchestration")
       pub slug: String,
       /// Absolute path to the feature's kitty-specs directory
       pub dir: PathBuf,
       /// Whether spec.md exists
       pub has_spec: bool,
       /// Whether plan.md exists
       pub has_plan: bool,
       /// Whether tasks/ directory has WP files
       pub has_tasks: bool,
   }
   
   /// Detect the target feature using multiple strategies.
   ///
   /// Priority: arg > branch > directory > None
   /// Returns None if no feature could be detected (caller should show selector).
   pub fn detect_feature(
       spec_prefix: Option<&str>,
       specs_dir: &Path,
   ) -> Result<Option<FeatureInfo>> {
       // 1. Try CLI argument first
       if let Some(prefix) = spec_prefix {
           if let Some(info) = detect_from_arg(prefix, specs_dir)? {
               tracing::info!(slug = %info.slug, "Feature detected from argument");
               return Ok(Some(info));
           }
       }
       
       // 2. Try git branch
       if let Some(info) = detect_from_branch(specs_dir)? {
           tracing::info!(slug = %info.slug, "Feature detected from git branch");
           return Ok(Some(info));
       }
       
       // 3. Try current directory
       if let Some(info) = detect_from_directory(specs_dir)? {
           tracing::info!(slug = %info.slug, "Feature detected from directory");
           return Ok(Some(info));
       }
       
       // 4. No detection — caller should show feature selector
       tracing::info!("No feature detected — selector needed");
       Ok(None)
   }
   ```

2. Add `pub mod detect;` to `crates/kasmos/src/launch/mod.rs`

### T030: Implement branch detection

**Purpose**: Parse the current git branch name for the `###-*` feature pattern, stripping any `-WP##` suffix.

**Steps**:

1. In `launch/detect.rs`, implement:
   ```rust
   use std::process::Command;
   
   /// Detect feature from current git branch name.
   ///
   /// Matches branches like "011-mcp-agent-swarm-orchestration" or
   /// "011-mcp-agent-swarm-orchestration-WP01" (strips WP suffix).
   fn detect_from_branch(specs_dir: &Path) -> Result<Option<FeatureInfo>> {
       let output = Command::new("git")
           .args(["rev-parse", "--abbrev-ref", "HEAD"])
           .output()?;
       
       if !output.status.success() {
           return Ok(None);
       }
       
       let branch = String::from_utf8_lossy(&output.stdout).trim().to_string();
       
       // Strip -WP## suffix if present
       let slug = strip_wp_suffix(&branch);
       
       // Validate it matches ###-* pattern
       if !is_feature_slug(&slug) {
           return Ok(None);
       }
       
       // Check if the feature directory exists
       resolve_feature_dir(&slug, specs_dir)
   }
   
   /// Strip -WP## suffix from a branch name.
   /// "011-foo-WP01" → "011-foo"
   /// "011-foo" → "011-foo" (unchanged)
   fn strip_wp_suffix(name: &str) -> String {
       // Pattern: -WP followed by digits at the end
       if let Some(idx) = name.rfind("-WP") {
           let suffix = &name[idx + 3..];
           if suffix.chars().all(|c| c.is_ascii_digit()) && !suffix.is_empty() {
               return name[..idx].to_string();
           }
       }
       name.to_string()
   }
   
   /// Check if a string matches the ###-* feature slug pattern.
   fn is_feature_slug(s: &str) -> bool {
       s.len() >= 4
           && s[..3].chars().all(|c| c.is_ascii_digit())
           && s.as_bytes()[3] == b'-'
   }
   ```

2. Add helper `resolve_feature_dir()`:
   ```rust
   /// Look up a feature slug in the specs directory.
   fn resolve_feature_dir(slug: &str, specs_dir: &Path) -> Result<Option<FeatureInfo>> {
       let dir = specs_dir.join(slug);
       if dir.is_dir() {
           Ok(Some(build_feature_info(slug.to_string(), dir)))
       } else {
           Ok(None)
       }
   }
   
   /// Build FeatureInfo by checking which artifacts exist.
   fn build_feature_info(slug: String, dir: PathBuf) -> FeatureInfo {
       FeatureInfo {
           has_spec: dir.join("spec.md").exists(),
           has_plan: dir.join("plan.md").exists(),
           has_tasks: dir.join("tasks").is_dir()
               && std::fs::read_dir(dir.join("tasks"))
                   .map(|entries| entries.filter_map(|e| e.ok())
                       .any(|e| e.file_name().to_string_lossy().starts_with("WP")))
                   .unwrap_or(false),
           slug,
           dir,
       }
   }
   ```

### T031: Implement argument resolution

**Purpose**: Match a spec prefix (e.g., "011") against kitty-specs/ entries.

**Steps**:

1. In `launch/detect.rs`:
   ```rust
   /// Detect feature from CLI argument (prefix match).
   ///
   /// "011" matches "011-mcp-agent-swarm-orchestration"
   /// "011-mcp" also matches (prefix of full slug)
   fn detect_from_arg(prefix: &str, specs_dir: &Path) -> Result<Option<FeatureInfo>> {
       let entries = list_feature_dirs(specs_dir)?;
       
       let matches: Vec<_> = entries
           .into_iter()
           .filter(|(slug, _)| slug.starts_with(prefix))
           .collect();
       
       match matches.len() {
           0 => Ok(None),
           1 => {
               let (slug, dir) = matches.into_iter().next().unwrap();
               Ok(Some(build_feature_info(slug, dir)))
           }
           _ => {
               // Multiple matches — return None, let caller handle disambiguation
               tracing::warn!(
                   prefix = prefix,
                   matches = matches.len(),
                   "Ambiguous feature prefix"
               );
               anyhow::bail!(
                   "Ambiguous prefix '{}' matches {} features: {}",
                   prefix,
                   matches.len(),
                   matches.iter().map(|(s, _)| s.as_str()).collect::<Vec<_>>().join(", ")
               );
           }
       }
   }
   
   /// List all feature directories in specs_dir.
   fn list_feature_dirs(specs_dir: &Path) -> Result<Vec<(String, PathBuf)>> {
       let mut features = Vec::new();
       
       if !specs_dir.is_dir() {
           return Ok(features);
       }
       
       for entry in std::fs::read_dir(specs_dir)? {
           let entry = entry?;
           let name = entry.file_name().to_string_lossy().to_string();
           if entry.file_type()?.is_dir() && is_feature_slug(&name) {
               features.push((name, entry.path()));
           }
       }
       
       features.sort_by(|a, b| a.0.cmp(&b.0));
       Ok(features)
   }
   ```
