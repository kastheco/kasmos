# WP13 — Dead Code Removal & Module Cleanup

## Goal

Remove all source files whose functionality has been replaced by the new `serve/`, `launch/`, and `setup/` modules. Clean up dangling module declarations and imports. Ensure clean compilation in both default and TUI feature configurations.

### T082 — Clean up dangling declarations

After all file removals:

1. Open `crates/kasmos/src/lib.rs` — remove ALL `pub mod` lines for deleted modules, remove ALL re-exports (`pub use`) for deleted items
2. Open `crates/kasmos/src/main.rs` — remove any `use kasmos::*` imports for deleted modules
3. Search entire `crates/kasmos/src/` for any remaining `use crate::` or `use kasmos::` referencing deleted modules:
   ```bash
   grep -rn "use crate::\(attach\|stop\|sendmsg\|cmd\|commands\|command_handlers\|engine\|detector\|health\|shutdown\|session\|start\|review_coordinator\)" crates/kasmos/src/
   ```
4. Fix any remaining references — either remove the import or update to use the new module path

### T083 — Verify compilation in both feature configurations

1. `cargo build -p kasmos` — must succeed with zero errors
2. `cargo build -p kasmos --features tui` — must succeed (TUI code behind feature flag compiles)
3. `cargo test -p kasmos` — all tests pass
4. `cargo clippy -p kasmos` — no warnings from remaining code (warnings from feature-gated code are acceptable)
5. Check for any `dead_code` warnings — these indicate additional cleanup needed

**If any compilation errors**:
- Identify the failing import or type reference
- Trace it back to a deleted module
- Either remove the dependent code (if it's also dead) or update the import to the new module path
- Re-run verification
