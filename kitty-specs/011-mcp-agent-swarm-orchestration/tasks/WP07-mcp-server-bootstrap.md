# WP07: MCP Server Bootstrap & Router

## Objective

Implement the `kasmos serve` MCP server skeleton using rmcp with stdio transport. Register tool stubs for all 9 tools and fully implement the 2 simplest tools (`list_features`, `infer_feature`). The server starts, accepts JSON-RPC 2.0 over stdio, and responds to `tools/list` with all 9 tool definitions.

## Context

### rmcp Overview

`rmcp` is the official Rust MCP SDK. Key patterns:
- `#[derive(ServerHandler)]` on a struct that holds server state
- `#[tool(name = "...", description = "...")]` proc macro on handler methods
- Input types derive `JsonSchema` (via `schemars`) for automatic schema generation
- Stdio transport via `rmcp::transport::stdio`
- Server runs via `rmcp::ServiceExt::serve` or similar

### T043: Define KasmosServer struct with shared state

**Purpose**: Create the server struct that holds all shared state needed by tool handlers.

**Steps**:

1. Create `crates/kasmos/src/serve/server.rs`:
   ```rust
   use crate::config::KasmosConfig;
   use super::registry::WorkerRegistry;
   use super::dashboard::DashboardState;
   use std::sync::Arc;
   use tokio::sync::Mutex;
   use std::sync::atomic::AtomicUsize;
   
   /// MCP server state shared across all tool handlers.
   pub struct KasmosServer {
       /// Runtime configuration
       pub config: KasmosConfig,
       
       /// In-memory worker registry (pane tracking)
       pub registry: Arc<Mutex<WorkerRegistry>>,
       
       /// Dashboard display state
       pub dashboard: Arc<Mutex<DashboardState>>,
       
       /// Last processed message index (for wait_for_event)
       pub msg_index: Arc<AtomicUsize>,
       
       /// Bound feature slug (if detected)
       pub feature: Arc<Mutex<Option<String>>>,
   }
   
   impl KasmosServer {
       pub fn new(config: KasmosConfig) -> Self {
           Self {
               config,
               registry: Arc::new(Mutex::new(WorkerRegistry::new())),
               dashboard: Arc::new(Mutex::new(DashboardState::default())),
               msg_index: Arc::new(AtomicUsize::new(0)),
               feature: Arc::new(Mutex::new(None)),
           }
       }
   }
   ```

### T045: Implement `list_features` tool (full)

**Purpose**: Fully implement the `list_features` MCP tool — scans kitty-specs/ for feature directories with status.

**Steps**:

1. Create `crates/kasmos/src/serve/tools/list_features.rs`:
   ```rust
   use schemars::JsonSchema;
   use serde::{Deserialize, Serialize};
   use crate::launch::detect;
   
   #[derive(Debug, Deserialize, JsonSchema)]
   pub struct ListFeaturesInput {}
   
   #[derive(Debug, Serialize)]
   pub struct ListFeaturesOutput {
       pub features: Vec<FeatureEntry>,
   }
   
   #[derive(Debug, Serialize)]
   pub struct FeatureEntry {
       pub slug: String,
       pub has_spec: bool,
       pub has_plan: bool,
       pub has_tasks: bool,
       pub wp_summary: Option<WpSummary>,
   }
   
   #[derive(Debug, Serialize)]
   pub struct WpSummary {
       pub total: usize,
       pub done: usize,
   }
   
   pub async fn handle(
       _input: ListFeaturesInput,
       specs_dir: &std::path::Path,
   ) -> ListFeaturesOutput {
       match detect::list_features(specs_dir) {
           Ok(features) => ListFeaturesOutput {
               features: features.into_iter().map(|f| {
                   // TODO: Count WPs and done status if has_tasks
                   FeatureEntry {
                       slug: f.slug,
                       has_spec: f.has_spec,
                       has_plan: f.has_plan,
                       has_tasks: f.has_tasks,
                       wp_summary: None, // TODO: parse task files for summary
                   }
               }).collect(),
           },
           Err(e) => {
               tracing::error!(error = %e, "Failed to list features");
               ListFeaturesOutput { features: vec![] }
           }
       }
   }
   ```

### T046: Implement `infer_feature` tool (full)

**Purpose**: Fully implement the `infer_feature` MCP tool — detects feature from environment.

**Steps**:

1. Create `crates/kasmos/src/serve/tools/infer_feature.rs`:
   ```rust
   use schemars::JsonSchema;
   use serde::{Deserialize, Serialize};
   use crate::launch::detect;
   
   #[derive(Debug, Deserialize, JsonSchema)]
   pub struct InferFeatureInput {}
   
   #[derive(Debug, Serialize)]
   pub struct InferFeatureOutput {
       pub detected: bool,
       pub feature_slug: Option<String>,
       pub method: String,  // "branch", "directory", "none"
   }
   
   pub async fn handle(
       _input: InferFeatureInput,
       specs_dir: &std::path::Path,
   ) -> InferFeatureOutput {
       // Try branch detection first
       if let Ok(Some(info)) = detect::detect_feature(None, specs_dir) {
           // detect_feature with None tries branch then directory
           return InferFeatureOutput {
               detected: true,
               feature_slug: Some(info.slug),
               method: "branch".to_string(), // TODO: track actual method
           };
       }
       
       InferFeatureOutput {
           detected: false,
           feature_slug: None,
           method: "none".to_string(),
       }
   }
   ```
