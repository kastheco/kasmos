## Scope Boundaries

**In scope**:
- Integration tests in `crates/kasmos/tests/` (new directory)
- Test fixtures in `crates/kasmos/tests/fixtures/`
- Documentation updates to quickstart.md, README.md, AGENTS.md
- Success criteria validation

**Out of scope**:
- Unit tests (each WP writes its own unit tests)
- End-to-end tests that spawn actual OpenCode agents (too heavyweight for automated testing)
- Performance benchmarking beyond basic timing assertions
- CI pipeline configuration (just mark Zellij-dependent tests as `#[ignore]`)

## Constraints

- Tests requiring Zellij MUST be marked `#[ignore]` — CI environments won't have Zellij running
- Timing tests use generous thresholds (2x spec requirement minimum) — hardware varies
- Integration tests should create temp directories and clean up after themselves
- Don't test OpenCode agent behavior — test kasmos's tool responses given known inputs
- Message parsing tests must use the exact `[KASMOS:{wp_id}:{EVENT}]` format — this is the protocol contract
